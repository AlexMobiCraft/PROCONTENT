# Brief: SM/Lead - План реализации мультиформатной ленты

## Контекст
Переход на мультиформатную ленту — это масштабный рефакторинг, затрагивающий API, UI и хранилище. По итогам обновления PRD и Архитектуры добавлены новые требования к нормализации БД (`post_media`), матрице доступов (RBAC), производительности и сложной UI-логике. Задача SM — организовать процесс так, чтобы не сломать текущий функционал, покрывая все новые ограничения и нефункциональные требования.

## План действий и задачи

1. **Декомпозиция Итерации**:
    - **Backend/DB (Step 1)**: Создание таблицы `post_media` (FK к `posts`). Обязательная настройка политик Row Level Security (RLS) в соответствии с матрицей RBAC (Автор создает/редактирует, Участница только читает).
    - **API (Step 1.1)**: Обновить `fetchPosts` и `fetchPostById` в сервисах. Использовать `JOIN` с новой таблицей `post_media`. Учесть SLA (NFR5): 95th percentile response time ≤ 500ms.
    - **UI Library (Step 2)**: Разработать Dumb UI компонент `GalleryGrid.tsx`. Отразить логику (FR16.1): 2-4 элемента (сетка), 5 (сетка 2х3), 6 (сетка 3х3), 7-10 (сетка 2х2 + карусель).
    - **Media Control (Step 3)**: Внедрить глобальный менеджер управления видео, чтобы гарантировать активное воспроизведение только одного видео в ленте (NFR4.1).
    - **SEO (Step 3.1)**: Настроить Open Graph `og:image` на основе первого или переданного (обложка) элемента галереи.
    - **Admin Tooling (Step 4)**: Обновить интерфейс загрузки медиа панели автора (до 10 файлов, выбор обложки).
    - **Telegram Migration (Step 5)**: Обновить скрипт импорта. Внедрить механизм "Exponential Backoff" со стейт-переходами (cursor) при Rate Limit от API Telegram без дублирования.

2. **Работа со Stories**:
    - Создать новые Stories под "Галереи и мультимедиа", "Менеджер воспроизведения видео", "Разделение прав доступа (RBAC)".
    - Пересмотреть Acceptance Criteria для существующих Story отображения ленты (проверки сеток, каруселей, NFRs производительности).

3. **QA & Testing**:
    - Обеспечить покрытие новыми юнит-тестами для компонента `GalleryGrid` (все вариации до 10 элементов).
    - Проверить сохранение позиции скролла при возврате из детального просмотра (учесть прыжки высоты постов).
    - Протестировать работу RLS-политик под разными JWT ролями.
    - Разработать тест-кейсы для симуляции ошибок Rate Limit при импорте из Telegram.

## Результат
План работ в формате задач для разработчиков, обновленные карточки историй и расширенная стратегия QA.
