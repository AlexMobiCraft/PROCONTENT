# Brief: SM/Lead - План реализации мультиформатной ленты

## Контекст
Переход на мультиформатную ленту — это масштабный рефакторинг, затрагивающий API, UI и хранилище. Задача SM — организовать процесс так, чтобы не сломать текущий функционал и обеспечить плавный переход.

## План действий и задачи

1. **Декомпозиция Итерации**:
    - **Backend/API (Step 1)**: Обновить `fetchPosts` и `fetchPostById` в `serverPosts.ts`. Использовать join с новой таблицей `post_media`.
    - **UI Library (Step 2)**: Создать базовый компонент `PostMediaGallery.tsx` в `src/components/media/`.
    - **Refactoring (Step 3)**: Обновить компоненты `PostCard.tsx` и `PostDetail.tsx`.
    - **Admin Tooling (Step 4)**: Обновить интерфейс загрузки медиа в панели администратора.

2. **Работа со Stories**:
    - Создать новую Story (например, 2.4 "Галереи и мультимедиа") или обновить 2.1/2.3.
    - Пересмотреть Acceptance Criteria для Story 2.3 (теперь пост может содержать несколько LCP элементов).

3. **QA & Testing**:
    - Обеспечить покрытие новыми юнит-тестами для компонента `PostMediaGallery`.
    - Проверить сохранение позиции скролла при возврате из детального просмотра (Story 2.3) с учетом изменившейся высоты постов.

## Результат
План работ в формате задач для разработчиков, обновленные карточки историй и стратегия тестирования.
