# Brief: Architect - Рефакторинг модели данных ленты (Multi-media)

## Контекст
Текущая схема БД (`posts.image_url`, `posts.type`) денормализована и не масштабируется для галерей. Необходимо нормализовать данные, выделив медиа-файлы в отдельную связанную сущность.

## Задачи для выполнения

1. **Проектирование схемы БД (Supabase)**:
    - Разработать таблицу `post_media`:
        - `id`: UUID (Primary Key)
        - `post_id`: UUID (FK to `posts.id`, CASCADE ON DELETE)
        - `media_type`: ENUM ('image', 'video')
        - `url`: TEXT (CDN link)
        - `thumbnail_url`: TEXT (для видео)
        - `order_index`: INTEGER (для сохранения порядка)
        - `is_cover`: BOOLEAN (флаг для SEO/OG-тэгов, если пользователь явно задал обложку)
    - **Ограничения (PRD):** Максимум 10 медиафайлов на один пост (Галерея).
    - **RLS / RBAC:** Реализовать Row Level Security по матрице доступов из PRD (только автор может создавать/редактировать `post_media`, участницы с активной подпиской могут только просматривать).
    - Подготовить SQL-скрипт миграции с переносом существующих `image_url` из старой таблицы.

2. **Архитектура Backend и интеграций**:
    - **Performance SLA (NFR5):** Оптимизировать запросы ленты (учесть JOIN с `post_media`) так, чтобы 95th percentile response time был ≤ 500ms.
    - **Telegram Миграция (NFR19.2):** Архитектура скрипта импорта должна группировать медиа без потерь данных и поддерживать "Exponential Backoff" со стейт-переходом (cursor) при Rate Limit ошибках от Telegram API.

3. **Слой представления и Типизация (Frontend)**:
    - Обновить `src/features/feed/types.ts`:
        - Добавить `PostMedia[]` в интерфейсы постов.
    - **UI Галерея (FR16.1):** Реализовать компонент (например, `GalleryGrid`), который рендерит медиа по правилам:
        - 2-4 элемента -> сетка
        - 5 элементов -> сетка 2х3
        - 6 элементов -> сетка 3х3
        - 7-10 элементов -> сетка 2х2, а остальные элементы ниже в формате карусели
    - **Управление Видео (NFR4.1):** Создать механизм контроля проигрывания, чтобы одновременно воспроизводилось не более одного видео.
    - **SEO:** Поддержка Open Graph `og:image` на основе превью-изображения.

## Ожидаемый результат
Готовый набор SQL-миграций и обновленные архитектурные документы, гарантирующие целостность данных после рефакторинга.
