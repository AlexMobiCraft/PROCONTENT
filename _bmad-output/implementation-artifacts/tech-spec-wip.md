# Tech-Spec: Перевод всего контента сайта на Словенский язык

**Created:** 2026-03-21T13:08:16+01:00

## Overview

### Problem Statement

Словенская аудитория сайта не имеет возможности читать контент и взаимодействовать с интерфейсом (кнопками, сообщениями, меню и т.д.) на родном языке. Текущие тексты, вероятно, захардкожены на одном языке (русском или английском), что делает сайт недоступным или неудобным для пользователей из Словении.

### Solution

Необходимо интегрировать надежное решение для интернационализации (i18n), подходящее для Next.js 16 App Router (предпочтительно `next-intl`), извлечь все захардкоженные строки интерфейса в локализационные словари (messages) и перевести их на словенский язык. Также потребуется добавить компонент для выбора и переключения языка пользователем.

### Scope

**In Scope:**
- Настройка `next-intl` (или аналогичного решения i18n) для Next.js 16 App Router с поддержкой словенского (sl) и текущего базового языка.
- Разделение всего статического текста (кнопки, лэйблы, заголовки, сообщения об ошибках, уведомления toast/sonner) и замена его на механизмы перевода (например `useTranslations` или `t()`).
- Создание файлов словаря (JSON) для словенского языка.
- Создание UI-компонента переключения языка (Language Switcher), интегрированного в хедер или настройки.

**Out of Scope:**
- Перевод динамического пользовательского контента, хранящегося в Supabase (посты, комментарии, профили пользователей) - если это требует изменения схемы БД, это выносится в отдельную задачу.
- Автоматический перевод пользовательского ввода.

## Context for Development

### Codebase Patterns

- Next.js 16 App Router - использование Server Components (RSC) по умолчанию, Client Components (с директивой `'use client'`) там, где необходима интерактивность.
- Tailwind CSS и Shadcn UI для стилизации - потребуется убедиться, что длинные словенские тексты не ломают верстку.

### Files to Reference

| File | Purpose |
| ---- | ------- |
| `src/app/layout.tsx` | Основной layout, требующий обертки провайдером i18n |
| `next.config.js` / `next.config.mjs` | Настройка конфигурации плагина локализации |

### Technical Decisions

- Использование `next-intl` рекомендуется из-за нативной поддержки RSC в Next.js App Router без принудительного рендеринга всего приложения на клиенте. 
- Настройки роутинга: стоит решить, будет ли использоваться path-based routing (`/sl/about`) или cookie-based routing без префикса (по умолчанию для `next-intl` часто рекомендуют path-based routing с middleware, но это может повлиять на существующие роуты).
