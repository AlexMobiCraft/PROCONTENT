# Tech-Spec: Быстрый перевод интерфейса сайта на словенский язык

**Created:** 2026-03-21

## Overview

### Problem Statement

Заказчику и целевой аудитории крайне необходимо, чтобы интерфейс сайта отображался на словенском языке прямо сейчас. Полноценная поддержка мультиязычности (i18n, next-intl) будет реализована позже. В настоящий момент нужен самый быстрый вариант: заменить все русские тексты в интерфейсе на словенские хардкодом.

### Solution

Заменить весь публичный текст (кнопки, лэйблы, заголовки, сообщения об ошибках) в React-компонентах и Zod-схемах с русского на словенский напрямую. Обновить язык тега `<html>` и SEO-метаданные.

### Scope

**In Scope:**
- Хардкодная замена русского текста в UI-компонентах (кнопки, модалки, навигация, футер).
- Перевод сообщений ошибок в Zod-схемах на словенский.
- Изменение локали для библиотек работы с датами (например, `date-fns` должна использовать `sl` вместо `ru`).
- Перевод `title` и `description` в файлах `layout.tsx` и `page.tsx`. Обязательно выставить `<html lang="sl">`.
- Обновление Email-шаблонов Supabase (сброс пароля, подтверждение почты) на словенский язык. Тексты шаблонов меняются прямо в Supabase Dashboard.

**Out of Scope:**
- Любая логика мультиязычности (next-intl, роутинг по локалям).
- Перевод пользовательского контента (содержимое постов из базы).
- **Перевод комментариев в исходном коде: они строго остаются на русском языке.** 

## Implementation Plan

### Tasks

- [x] Task 1: Глобальный поиск и замена UI-текстов
  - Обойти все файлы в `src/app` и `src/components`. Найти русские тексты и перевести их на словенский язык.
  - **Критично:** Переводить необходимо не отдельными словами, а **целыми фразами** с учетом контекста, чтобы словенский язык звучал естественно (адекватный смысловой перевод интерфейса).
- [x] Task 2: Перевод Zod-схем и ошибок валидации
  - Zod-схем в проекте нет. Все валидации выполняются нативно в формах (LoginForm, RegisterForm, ForgotPasswordForm, UpdatePasswordForm) — все сообщения об ошибках переведены на словенский.
- [x] Task 3: Обновление SEO и тега HTML

  - `date-fns` не используется. `Intl.DateTimeFormat` (`toLocaleDateString`) переключён с `'ru-RU'` на `'sl-SI'` в `SubscriptionCard.tsx`.
