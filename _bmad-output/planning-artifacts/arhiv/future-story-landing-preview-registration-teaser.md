# Future Story Idea: teaser preview на лендинге с CTA `Registriraj se`

## Статус

Идея для отдельной будущей истории.
Не входит в scope Story 4.3.

## Контекст

В рамках обсуждения Story 4.3 было решено не смешивать два разных сценария:

- публичный лендинг для анонимного посетителя
- onboarding для новой авторизованной участницы

Для лендинга предложен более простой сценарий, чем открытие детальной страницы поста или использование onboarding-логики.

## Идея

При клике анонимного посетителя на preview-пост на лендинге приложение не открывает полный пост и не ведёт пользователя в onboarding.

Вместо этого пользователь видит teaser-карточку поста с ограниченным содержимым и CTA на регистрацию.

## Предлагаемый UX-сценарий

- Пользователь открывает публичный лендинг.
- В секции preview нажимает на один из постов.
- Открывается teaser-вариант карточки поста.
- Полный контент поста не показывается.
- В teaser-карточке отображается сообщение на словенском языке:
  - `Za ogled vseh objav se registriraj`
- Ниже отображается CTA-кнопка:
  - `Registriraj se`

## Черновик acceptance criteria для будущей истории

1. **Teaser вместо полного поста для анонимного посетителя:**
   **Given** анонимный посетитель открывает лендинг
   **When** он нажимает на preview-пост
   **Then** приложение показывает teaser-карточку вместо полного контента поста
   **And** teaser содержит только безопасный ограниченный набор данных
   **And** полный контент недоступен без регистрации

2. **CTA на регистрацию:**
   **Given** teaser-карточка открыта
   **When** пользователь видит блок призыва к действию
   **Then** отображается текст `Za ogled vseh objav se registriraj`
   **And** отображается кнопка `Registriraj se`
   **And** кнопка ведёт на маршрут регистрации / входа, принятый в проекте

3. **Сохранение продуктовых ограничений:**
   **Given** функция реализуется в публичной части приложения
   **When** разработчик проектирует data flow
   **Then** анонимный доступ не раскрывает полное содержимое `posts`
   **And** UI остаётся на словенском языке
   **And** решение не ломает существующую секцию preview на лендинге

## Архитектурные заметки

- Этот сценарий должен использовать безопасный публичный data path.
- Нельзя открывать анонимный доступ ко всей таблице `posts`.
- Для teaser нужно отдавать только поля, реально нужные для preview / teaser UI.
- Это отдельный продуктовый сценарий и его лучше не смешивать с логикой `/onboarding`.

## Связь со Story 4.3

- Story 4.3 остаётся про:
  - управление `is_landing_preview`
  - управление `is_onboarding`
  - управление `whatsapp_url`
  - обновление onboarding-потока для авторизованной новой участницы

- Будущая история может использовать результаты 4.3, но не должна менять её scope задним числом.

## Возможное рабочее название будущей истории

- `Teaser access for anonymous visitors on landing`
