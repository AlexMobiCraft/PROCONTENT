# Техническая спецификация: Динамическая синхронизация категорий

## Проблема
Список категорий в компонентах `TopicsPanel` (правый сайдбар) и `CategoryScroll` (фильтр в ленте) захардкожен на фронтенде. Когда администратор создает новую категорию через `CategoryManager`, эти списки не обновляются, так как они не подключены к базе данных.

## Предлагаемые изменения

### 1. Глобальное состояние (`src/features/feed/store.ts`)
- Добавить массив `categories: Category[]` в `FeedState`.
- Добавить экшен `setCategories: (categories: Category[]) => void`.

### 2. Загрузка данных и гидратация (`src/features/feed/components/FeedPageClient.tsx`)
- В `useEffect` (при монтировании страницы ленты) добавить вызов `getCategories()` из `@/features/admin/api/categories`.
- Сохранять результат в `useFeedStore` через `setCategories`.
- Это обеспечивает наличие актуальных категорий при входе в ленту.

### 3. Обновление компонентов (Динамический рендеринг)
- **`TopicsPanel.tsx`**: Заменить статический массив `topics` на селектор из `useFeedStore`.
- **`CategoryScroll.tsx`**: Аналогично заменить статический массив `topics` на данные из стора.
- *Примечание:* Сохранить специальные категории, такие как «Все» (All), если их нет в базе.

### 4. Ревалидация при навигации админа
- Так как администратор может добавить категорию и затем перейти в ленту, `useEffect` в `FeedPageClient` обеспечит получение свежих данных.

## План реализации

### Шаг 1: Обновление Zustand-стора
Добавить поле и экшен в `src/features/feed/store.ts`.

### Шаг 2: Загрузка в FeedPageClient
Добавить логику fetch-запроса при инициализации ленты.

### Шаг 3: Рефакторинг UI-компонентов
Удалить хардкод из `TopicsPanel` и `CategoryScroll`, подписав их на изменения в `useFeedStore`.

## Критерии приемки
1. Администратор создает категорию "#новое" в `/admin/categories`.
2. При переходе в ленту `/feed` категория "#новое" отображается в правом блоке.
3. При переходе в ленту `/feed` на мобильном устройстве категория "#новое" отображается в меню фильтров.
4. Удаление категории в админке также корректно отражается в ленте после обновления страницы.
