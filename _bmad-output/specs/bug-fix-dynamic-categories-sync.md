# Техническая спецификация: Динамическая синхронизация категорий

## Intent

**Problem:** Список категорий в ленте и сайдбаре захардкожен. Когда админ создает категорию в панели управления, она не появляется в ленте без ручного вноса в код. Это ломает пользовательский опыт администратора.

**Approach:** Перевести компоненты на динамическое получение данных из Zustand-стора, который будет наполняться при монтировании страницы ленты. Добавить обработку состояний загрузки (скелетоны) и ошибок сетевых запросов.

## Boundaries & Constraints

**Always:**
- Использовать существующий Zustand-стор `useFeedStore`.
- Сохранять системную категорию "VSE" (All) как первичную.
- Использовать компоненты из `@/components/ui/skeleton` для индикации загрузки.

**Ask First:**
- Если структура данных `Category` из API изменится (сейчас ожидаем `{id, name, slug}`).

**Never:**
- Не делать прямых SQL-запросов из UI-компонентов (только через API/Store).
- Не использовать сторонние библиотеки для стейта, кроме тех, что уже есть (Zustand).

## Code Map

- `src/features/feed/store.ts` -- Глобальное состояние для категорий и статуса загрузки.
- `src/features/feed/components/FeedPageClient.tsx` -- Контроллер загрузки данных и обработки жизненного цикла.
- `src/components/feed/CategoryScroll.tsx` -- Компонент фильтров (лента + мобильное меню).
- `src/features/feed/components/TopicsPanel.tsx` -- Компонент сайдбара (десктоп).

## Tasks & Acceptance

- [x] `src/features/feed/components/FeedPageClient.tsx` -- Реализовать безопасный fetch с обработкой ошибок и очисткой.

- [x] `src/features/feed/components/TopicsPanel.tsx` -- Внедрить скелетоны при загрузке.

**Acceptance Criteria:**
- [x] **Given** лента загружается, **when** данные еще не получены, **then** отображаются скелетоны в фильтрах и сайдбаре.
- [x] **Given** ошибка загрузки API, **when** запрос завершается, **then** пользователь видит `toast.error`, а скелетоны скрываются.
- [x] **Given** категория "VSE" выбрана, **when** смотрим на мобильную иконку, **then** она НЕ подсвечивается как активная тема из БД.
- [x] **Given** админ создал тему, **when** переходит в ленту, **then** название темы появляется в списке без перезагрузки браузера.
