# Переход с test на live Stripe-аккаунт

**Применимо:** только для смены Stripe-аккаунта, Supabase не меняется.

## Шаги

### 1. Получить ключи от live-аккаунта Stripe

**Вход в Stripe Dashboard:**
1. Перейдите на https://dashboard.stripe.com
2. Войдите под аккаунтом владельца бизнеса
3. **Важно:** убедитесь что вы в правильном аккаунте (название компании вверху)

**Переключение на Live режим:**
1. В левом верхнем углу найдите переключатель режимов
2. Нажмите на него и выберите **Live** (вместо Test)
3. Интерфейс станет оранжевым (Live) вместо синего (Test)

**Получение API ключей (новый интерфейс):**

**Способ 1 - через меню:**
1. Нажмите **More** в левом меню (внизу)
2. В разделе **Developers** выберите **API keys**

**Способ 2 - через настройки:**
1. Нажмите на ваш **аватар/название компании** в правом верхнем углу
2. Выберите **Account settings**
3. В левом меню настроек найдите **API keys**

**Способ 3 - прямой URL:**
- Перейдите на https://dashboard.stripe.com/apikeys

**Действия с API ключами:**
1. Вы увидите список ключей. Если нет → нажмите **Create secret key**
2. Нажмите **Reveal** чтобы увидеть полный ключ
3. **Скопируйте `STRIPE_SECRET_KEY`** - начинается с `sk_live_`
4. **Сохраните в безопасном месте** - больше не сможете его увидеть полностью

**Проверка ключа:**
- Должен начинаться с `sk_live_` (не `sk_test_`)
- Длина около 150 символов
- Содержит буквы, цифры и подчеркивания

### 2. Создать webhook endpoint

**Создание webhook (новый интерфейс):**

**Способ 1 - через меню:**
1. Нажмите **More** в левом меню (внизу)
2. В разделе **Developers** выберите **Webhooks**
3. Нажмите **Add endpoint** (или **Create endpoint**)

**Способ 2 - прямой URL:**
- Перейдите на https://dashboard.stripe.com/webhooks

**Создание endpoint:**
1. Нажмите **Add destination** (в правом верхнем углу)
2. **Endpoint URL:** введите `https://ваш-домен.com/api/webhooks/stripe`
   - Замените `ваш-домен.com` на реальный домен приложения
3. **HTTP method:** оставьте `POST`
4. Нажмите **Add destination** (или **Create endpoint**)

**Настройка событий:**
1. После создания endpoint нажмите на него
2. В разделе **Events** нажмите **Add events**
3. Выберите эти события:
   - `checkout.session.completed` - успешная оплата
   - `customer.subscription.updated` - изменение подписки
   - `customer.subscription.deleted` - отмена подписки
4. Нажмите **Add events** → **Save**

**Получение webhook secret:**
1. В настройках webhook найдите раздел **Signing secret**
2. Нажмите **Reveal** чтобы увидеть секрет
3. **Скопируйте `STRIPE_WEBHOOK_SECRET`** - начинается с `whsec_`
4. **Сохраните в безопасном месте**

**Проверка webhook:**
- Должен начинаться с `whsec_`
- Длина около 50 символов
- Нужен для безопасности webhook запросов

### 3. Скопировать live цены продуктов

**Поиск продуктов:**
1. В левом меню → **Products** → **Products**
2. Вы увидите список ваших продуктов/тарифов
3. Нажмите на нужный продукт

**Получение ID цен:**
1. Внутри продукта найдите раздел **Pricing**
2. Вы увидите список цен (например: Monthly, Quarterly)
3. Для каждой цены нажмите **Copy ID** или скопируйте вручную
4. **ID цены** начинается с `price_` (например: `price_1ABC...`)

**Что нужно скопировать:**
- `STRIPE_MONTHLY_PRICE_ID` - ID для ежемесячной подписки
- `STRIPE_QUARTERLY_PRICE_ID` - ID для квартальной подписки

**Проверка ID цен:**
- Должны начинаться с `price_`
- Длина около 50-70 символов
- Содержат буквы, цифры и подчеркивания

**Важно:** Убедитесь что вы в **Live** режиме при копировании ID!

### 4. Обновить переменные окружения

**Локально** в `.env.local`:
```
STRIPE_SECRET_KEY=sk_live_...
STRIPE_WEBHOOK_SECRET=whsec_...
STRIPE_MONTHLY_PRICE_ID=price_...
STRIPE_QUARTERLY_PRICE_ID=price_...
```

**На хостинге** (Vercel, если используется):
- Settings → Environment Variables → обновить эти 4 переменные
- Передеплоить проект

### 5. Синхронизировать существующих клиентов

После смены ключей запустить скрипт:
```bash
npm install -D tsx dotenv  # если не установлены
npx tsx scripts/sync-stripe-subscriptions.ts --dry-run  # проверка
npx tsx scripts/sync-stripe-subscriptions.ts            # применить
```

Скрипт автоматически:
- Получит все активные подписки из live-Stripe
- Найдёт профили в Supabase по email
- Обновит `stripe_customer_id`, `stripe_subscription_id`, `subscription_status`, `current_period_end`

**Что делать с "не найденными"** (клиенты в Stripe, но не в БД):
- Если email отличается → обновить вручную через Supabase SQL Editor или скрипт

**Результат:** все 29 клиентов будут привязаны к их Stripe-подпискам в БД.
